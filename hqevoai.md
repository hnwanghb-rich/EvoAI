## 一、项目背景与目标

### 1.1 背景

合群汽车销售集团旗下拥有多个4S门店，业务覆盖新车销售、维修技术服务、售后客服三大板块。一线员工（销售顾问、维修技师、售后客服）在日常工作中积累了大量的实战经验，但这些经验目前处于：
- **散落状态**：存在师傅脑子里、纸质笔记、微信聊天记录中，无法沉淀复用
- **流失状态**：核心员工离职，经验随之流失
- **孤岛状态**：销售、维修、客服三体系各自为战，无法交叉赋能

### 1.2 建设目标

构建一个 **AI驱动的企业级知识库系统**，实现：
1. 将散落的业务经验标准化、数字化入库
2. 通过数字老师"阿能"为全员提供24小时知识检索与辅导
3. 建立"知识沉淀→实战验证→迭代进化"的自我进化飞轮
4. 支持手机端/PC端/Pad端/BI大屏多终端使用
5. 支持50人同时并发，页面响应 <0.5秒

## 二、核心设计理念

### 2.1 三位一体知识飞轮

```
                    ┌──────────────────────────────┐
                    │     公共标准化知识底座        │
                    │  (车型参数/政策/质保/维修规范) │
                    └──────────────┬───────────────┘
           ┌───────────────────────┼───────────────────────┐
           ▼                       ▼                       ▼
   ┌───────────────┐     ┌───────────────┐     ┌───────────────┐
   │ 销售业务飞轮  │◄───►│ 技术维修飞轮  │◄───►│ 售后客服飞轮  │
   │ 话术/竞品/谈判│     │ 通病/故障/经验│     │ 沟通/续保/投诉│
   └───────┬───────┘     └───────┬───────┘     └───────┬───────┘
           └───────────────────────┼───────────────────────┘
                                   ▼
                    ┌──────────────────────────────┐
                    │      企业自我进化中枢        │
                    │  经验萃取 → 标准化 → 全员赋能 │
                    └──────────────────────────────┘
```

### 2.2 系统边界（重要）

本系统**只管业务知识经验，不负责实现业务流程**：
- ✅ 管理：销售话术、维修技术经验、客服沟通技巧、政策法规知识
- ✅ 提供：知识检索、AI问答、经验沉淀、学习考核、能力分析
- ❌ 不负责：CRM销售流程、维修工单流程、售后派单流程等业务系统功能

### 2.3 核心逻辑

**知识标准化 → 经验场景化 → 实战验证化 → 数据迭代化 → 全员能力升级 → 业务增长 → 反哺知识体系**

### 4.2 各类数据处理方案

#### 4.2.1 结构化文档（PDF/Word）处理

- 使用 Python Unstructured / PyMuPDF 提取文本
- 保留文档层级结构（标题→章节→段落）
- 自动分块：500 tokens/块，重叠50 tokens
- 元数据标记：文件名、来源目录、页码

#### 4.2.2 表格数据（Excel）处理

- 每行数据按模板转为自然语言描述段落
- 示例：`"车型：星瑞 i-HEV，动力：智擎混动，核心卖点：..."`优于 `"price:18.5"`
- 保留原始文件链接供追溯

#### 4.2.3 视频处理（培训视频/看板说明）

- Whisper medium/large-v3 进行语音识别（中文）
- 每30秒或200字切分为一个知识片段
- 保留 `[start_sec, end_sec, jump_url]` 元数据
- 前端数字人回答时可跳转到视频对应时间点

#### 4.2.4 语音留言处理（核心功能）

- 师傅通过手机端小程序呼叫数字老师阿能
- 语音录入经验 → Whisper实时转文字 → 生成经验草稿
- 师傅确认/修改草稿 → 提交审核 → 管理员审核通过后入库
- 记录积分（提交+1分，被采用+10分）

#### 4.2.5 实时语音问答

- 用户在手机端语音提问 → Whisper实时转文字 → RAG检索 → LLM生成答案 → TTS语音播报
- 支持降噪处理（车间、展厅嘈杂环境）

### 7.1 知识掌握度评估模型

对每条知识，系统记录每位员工是否：已阅读 / 已学习 / 已通过测试 / 已在工作中使用。

## 九、技术架构方案

### 9.1 技术选型（最简方案）

| 层级 | 技术选型 | 说明 |
|------|---------|------|
| 前端(手机) | 微信小程序原生 / Uni-App | 跨端兼容 |
| 前端(PC/Pad) | Vue 3 + Element Plus | 轻量，生态成熟 |
| 前端(BI大屏) | Vue 3 + ECharts + DataV | 数据可视化 |
| 后端API | FastAPI (Python 3.11+) | 高性能异步，学习成本低 |
| 关系数据库 | PostgreSQL 16 | 结构化数据存储 |
| 缓存 | Redis | 热点数据缓存，会话管理 |
| 向量数据库 | Milvus (standalone) | 向量存储与检索 |
| 向量模型 | bge-large-zh-v1.5 | 中文语义理解最佳 |
| 语音识别 | Whisper large-v3 | 本地部署，中文优化 |
| LLM | Qwen2.5-7B-Instruct (vLLM部署) | 私有化，数据安全 |
| 文件存储 | MinIO / 本地文件系统 | 对象存储，兼容S3 |
| 消息队列 | Redis Stream / 内存队列 | 异步任务（视频处理） |
| 部署 | Docker Compose 单机部署 | 最简部署方案 |

### 9.2 为什么选这个方案

1. **全部可私有化部署**：数据不出企业内网，满足数据安全要求
2. **Python全栈后端**：FastAPI + vLLM + Whisper + Milvus客户端 全部Python生态，无需多语言切换
3. **Docker Compose一键部署**：PostgreSQL + Redis + Milvus + MinIO + FastAPI + vLLM，6个容器即完整系统
4. **50并发无压力**：FastAPI异步 + Redis缓存 + Milvus索引，轻松满足
5. **低运维成本**：无需K8s，单机（32GB内存 + GPU可选）即可运行

### 9.3 部署架构（Docker Compose）

```
┌─────────────────────────────────────────────────┐
│                 Nginx (反向代理)                  │
│              port: 80 / 443                      │

      │       │       │       │
      ▼       ▼       ▼       ▼
┌─────────┐ ┌───────┐ ┌───────┐ ┌─────────────────┐
│ FastAPI │ │  Vue  │ │vLLM  │ │  Whisper Worker │
│ :8000   │ │Static │ │:8001 │ │  (后台异步)      │
│         │ │ Files │ │(GPU) │ │                 │
└──┬──┬───┘ └───────┘ └───────┘ └─────────────────┘
   │  │
   ▼  ▼
┌────────┐ ┌────────┐ ┌────────┐ ┌────────┐
│PostgreSQL│ │ Redis  │ │Milvus │ │ MinIO  │
│  :5432   │ │ :6379  │ │:19530 │ │ :9000  │
└────────┘ └────────┘ └────────┘ └────────┘
```

### 9.4 性能保障措施

- **Redis缓存**：热点知识条目、用户会话、排行榜数据
- **PostgreSQL索引优化**：knowledge_entries 的 category_id、status、car_brand 建立联合索引
- **Milvus索引**：IVF_FLAT + 1024 nlist，百万级数据毫秒级检索
- **分页查询**：列表类接口全部分页，每页20条
- **CDN/静态资源**：视频文件通过MinIO提供，前端静态资源Nginx直接serve
- **连接池**：SQLAlchemy连接池20连接，Redis连接池50连接
- **API响应时间保证**：<0.5s（查询类），<2s（AI生成类）

## 十、飞轮进化机制

### 10.1 每周萃取

- 每周末系统自动统计本周各岗位提交的经验数
- 按"被浏览次数"和"有用反馈数"排序
- 推送TOP5优质经验给管理员，建议置顶推广

### 10.2 每月迭代

- 每月1号系统自动标记：超过6个月未更新的政策类知识为"待更新"
- 6个月内被浏览次数<5的经验类知识标记为"低效"，建议淘汰归档
- 管理员审核确认后批量处理

### 10.3 数据驱动筛选

- **有效经验**：被浏览次数>10 且 有用率>70% → 自动置顶推荐
- **待优化经验**：被浏览次数>10 但 有用率<30% → 建议修订
- **无效经验**：被浏览次数<3 且留存超6个月 → 建议归档

### 10.4 积分激励

| 行为 | 积分 | 说明 |
|------|------|------|
| 提交经验 | +1 | 每次有效提交 |
| 经验被采用入库 | +10 | 审核通过 |
| 经验被标记"有用" | +2 | 每次被同事标记有用 |
| 经验进月度TOP5 | +50 | 额外奖励 |
| 每日一题答对 | +1 | 每题 |
| 完成必修课程 | +5 | 每课 |

系统实时显示积分排行榜，公司根据积分排名给予物质奖励。
