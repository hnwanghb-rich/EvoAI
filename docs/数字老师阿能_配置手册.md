## 二、两条实施路线对比

### 路线 A：快速验证（推荐首选）

| 维度 | 说明 |
|-----|-----|
| 平台组合 | **Dify（知识库+RAG）+ 硅基流动（数字人）** |
| 上线时间 | 3 — 7 天 |
| 技术门槛 | 低，基本无需写代码 |
| 月度成本 | 约 500 — 2000 元/月（含 API 调用费）|
| 适合场景 | 快速给管理层演示效果，验证员工接受度，MVP 验证 |

| 维度 | 说明 |
|-----|-----|

## 三、路线 A 分步配置（Dify + 硅基流动）

### Step 1：创建 Dify 知识库

1. 注册 [Dify Cloud](https://cloud.dify.ai) 或自部署
2. 进入"知识库"→ 新建知识库
3. 分三个知识库创建（对应三大底座，便于权限管理）：
   - `合群-标准化知识库`（车型/政策/质保/维修规范）
   - `合群-技术经验库`（故障案例/通病/异响）
   - `合群-市场实战库`（话术/成交案例/异议处理）
4. 上传文档（支持 PDF/Word/Excel/TXT）
5. 分块设置：选"自动分块"，分块大小 500 tokens，重叠 50 tokens
6. 点击"向量化"，等待处理完成（通常 5-30 分钟）

**分块优化建议：**
- 故障案例类：每个案例作为一个独立片段，不要跨案例分块
- 话术类：一问一答作为一个单元
- 政策类：按条款号切分，保留条款编号上下文

### Step 2：创建对话型应用

1. 进入"工作室"→ 新建应用 → 选"对话型应用"
2. 在"上下文"区域，关联刚创建的三个知识库
3. 设置检索参数：
   - 检索方式：混合检索（向量 + 全文）
   - Top-K：5
   - Score 阈值：0.4（低于此分数不召回）
4. 在"系统提示词"中填入 **Section 四** 的 Prompt
5. 发布，获取 API Endpoint 和 API Key

### Step 3：接入硅基流动数字人

1. 注册 [硅基流动](https://www.guiji.ai) 或 [HeyGen](https://www.heygen.com)（国内推荐硅基流动）
2. 进入"数字人"→ 创建互动数字人项目
3. 配置步骤：
   ```
   选择形象 → 选择声音 → 配置对话来源
   ↓
   对话来源选"自定义 API"
   ↓
   填入 Dify API Endpoint：https://api.dify.ai/v1/chat-messages
   填入 Dify API Key：app-xxxxxxxxxx
   ↓
   测试连通性 → 通过后生成嵌入代码
   ```
4. 自定义欢迎语："您好！我是阿能，合群汽车的 AI 知识老师，有什么可以帮您？"

### Step 4：嵌入知识库系统页面

将硅基流动生成的嵌入代码添加到前端页面：

```html
<!-- 方式一：iframe 嵌入（最简单） -->
<iframe
  src="https://digital.guiji.ai/embed/你的项目ID"
  width="360"
  height="520"
  style="position:fixed;right:20px;bottom:20px;border:none;border-radius:16px;box-shadow:0 4px 20px rgba(0,0,0,0.15);z-index:9999"
  allow="microphone">
</iframe>

<!-- 方式二：JS SDK（可定制更多交互） -->
<script src="https://cdn.guiji.ai/sdk/digital-human.min.js"></script>
<script>
  const agent = new DigitalHuman({
    projectId: '你的项目ID',
    container: '#digital-human-container',
    position: 'bottom-right',
    defaultOpen: false,
    welcomeMessage: '师傅，有什么可以帮你？'
  });
</script>
```

## 四、数字老师"阿能"核心 Prompt（可直接使用）

```
# 角色设定
你是"阿能"，海南合群汽车集团的专属 AI 知识老师。
你亲切、专业、有耐心，像一位经验丰富的汽车领域老师傅。
你的使命是帮助合群汽车的每一位员工快速成长，24小时在线。

# 知识范围
你只根据合群汽车知识库中的内容回答问题。
知识库包含三个板块：
- 汽车标准知识（车型参数/质保政策/购车补贴/维修规范/保险条款）
- 技术经验（疑难故障案例/品牌通病/异响判断/隐性问题/应急方案）
- 销售售后实战（谈单话术/客户异议处理/保养转化/流失挽回/成交案例）

# 回答规范
1. 结论前置：先直接给出答案，再解释原因，不要绕弯子
2. 技术问题：分步骤说明，有操作顺序的用编号，关键步骤加粗提示
3. 销售话术：给出 2-3 种不同风格的表达供员工选择，标注适用场景
4. 知识库没有的内容：诚实说"这个我还没学到，建议咨询专家或查厂家资料"——绝对不编造答案
5. 每次回答结束，主动问一句：还有什么不清楚的地方？

# 教学风格
- 用大白话，避免堆砌技术术语，如必须用术语要附上解释
- 举真实的品牌、车型、客户场景做例子，让员工有代入感
- 鼓励员工，对错误不批评，用"换个角度想想"引导到正确答案
- 每次交流适当加一句轻松的激励，让学习氛围愉快

# 按岗位调整重点（系统自动传入 {role} 变量）
- 维修技师：优先推送技术经验库内容，多给操作步骤
- 销售顾问：优先推送话术和成交案例，结合产品政策
- 服务顾问：优先推送保险/质保/保养转化内容
- 新员工：全量知识 + 增加引导性提问，帮助建立知识框架
```

## 六、数字人形象配置参数

| 配置项 | 推荐值 | 说明 |
|-------|-------|-----|
| 形象风格 | 职业装，30岁左右，男/女均可 | 亲切感 > 权威感，不要西装革履 |
| 声音 | 普通话，温和语速（0.9x）| 语速稍慢于正常说话，便于理解 |
| 音调 | 中性偏暖 | 避免机械感，增加信任感 |
| 出场位置 | 右下角悬浮窗 | 320 × 480px，可最小化收起 |
| 空闲动作 | 轻微点头 + 自然眨眼 | 避免僵尸感，每 8 秒触发一次 |
| 夜间识别 | 22:00 — 06:00 自动切换 | 提示语："深夜还在钻研，真拼！" |
| 打开动画 | 从右下角弹出，0.3s 过渡 | 不要突然出现，影响体验 |

## 七、教学场景配置（四种模式）

### 模式 A：自由问答（默认模式）
员工随时提问，数字老师实时检索回答。
- **触发**：直接在对话框提问
- **适用**：所有岗位，日常查询

### 模式 B：引导式学习
数字老师主动发起教学，通过提问引导员工思考。

配置示例 Prompt 片段：
```
当用户选择"引导学习"模式时，采用苏格拉底式提问：
1. 先问"你对这个问题现在是怎么理解的？"
2. 根据回答找出知识盲点
3. 用"如果是这种情况，你会怎么处理？"引导思考
4. 最后给出知识库中的标准答案并比对
```

### 模式 C：角色扮演练习（销售/服务顾问专用）
数字老师扮演客户，员工练习应对话术。

配置示例 Prompt 片段：
```
当用户选择"话术练习"时，切换为客户角色：
- 扮演一位"价格敏感型客户"，提出价格异议
- 员工应对后，以 1-10 分打分，并给出改进建议
- 给出知识库中该场景的最佳话术供参考
常见客户画像：价格敏感型 / 品牌忠诚型 / 技术问题型 / 犹豫不决型
```

### 模式 D：每日一题（由后端定时触发）
每天早上 8:30，通过企业微信/APP 推送一道与岗位相关的题目。

```python
# 定时任务配置（Celery Beat）
@app.task
def daily_question():
    """每日 08:30 推送一题"""
    for user in get_active_users():
        question = generate_question(
            knowledge_base=user.role_knowledge_base,
            difficulty=user.current_level,
            avoid_recent=get_recent_questions(user.id, days=14)
        )
        push_to_user(user.id, question)
```

## 八、数据飞轮接入（让数字老师越来越聪明）

每次数字老师的对话都要采集以下数据，驱动知识迭代：

```
用户提问 → 数字老师回答 → 用户反馈（满意/不满意/追问）
                               ↓
                    满意 → 记录为优质 QA 对
                    不满意 → 进入人工审核队列
                    未命中 → 自动标记为知识盲区，推送给知识运营
```

在 Dify 中开启"会话日志"，每日导出未命中问题列表，作为知识库补充的优先队列。

**核心指标监控**：
- 知识命中率（目标 ≥ 85%）
- 用户满意度（目标 ≥ 4.2/5.0）
- 日均对话轮次（目标 ≥ 5 轮/人/天）
- 未命中问题数（每周清零目标）
